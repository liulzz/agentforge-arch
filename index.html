SaaS 整体模式 vs 子产品独立模式

SaaS 整体模式

四层能力完整集成在一个平台实例中，数据完全打通。适合需要一站式解决方案的中大型企业客户。统一控制台、统一计费、统一用户体系。

子产品独立模式

四大能力面拆分为独立子产品，可单独部署、独立计费。客户按需选配，通过 Kafka 事件总线实现跨产品数据同步。适合模块化采购场景。

🛠 低代码平台 · AgentDev Studio
ai-lcap-Server
描述：低代码平台的服务端核心引擎。基于 Python 构建，提供元数据驱动、模型解析、流程执行能力。负责数据处理与逻辑运算。
构建语言：Python  |  编译构建：Python  |  仓库语言：Python
ai-lcap-web
描述：低代码平台的 Web 控制台与设计器。基于 Vue 3 + TypeScript + Element Plus + Vite 构建，提供可视化拖拽编排、表单设计与流程画布能力。

⚙️ 运行平台 · AgentRuntime
ai-arp-Server
描述：Agent 运行平台的服务端核心引擎。基于 Java + Spring Cloud 构建，提供 Agent 注册发现、会话管理、沙箱隔离、路由分发能力。承载 Agent 全生命周期运行时态管理。
构建语言：Java  |  编译构建：Maven  |  仓库语言：Java

命名规则

• 统一前缀：ai- 表示 AgentForge 平台体系
• 后端后缀：-Server（大写 S，Java Maven 工程规范）
• 前端后缀：-web（小写，npm 包名规范）
• 缩写取自英文全称：lcap = Low-Code Application Platform, arp = Agent Runtime Platform, aep = Agent Evaluation Platform, aeco = Agent Ecosystem Platform
