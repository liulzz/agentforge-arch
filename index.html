总体架构

AgentForge 采用「中台 + 子产品」双模式设计：SaaS 模式提供完整中台能力，同时每个能力域可作为独立子产品单独交付。

核心设计原则：每个能力域（开发/运行/评测/生态）既可作为 SaaS 中台的一部分整体交付，也可作为独立子产品单独销售。子产品之间通过标准 API 和事件总线解耦。

SaaS 整体 vs 子产品独立 — 产品形态对比

SaaS 整体 AgentForge Platform

  • 一站式平台：开发 + 运行 + 评测 + 生态全能力
  • 统一体验：一个控制台管理所有 Agent 生命周期
  • 数据打通：开发数据 → 运行指标 → 评测结果全链路
  • 计费：按用量统一计费（API 调用 + 计算 + 存储）
  • 适合：需要完整 Agent 能力的中大型团队

子产品独立 按需购买

  • 灵活组合：只买需要的能力域
  • 独立交付：每个子产品有独立的入口和计费
  • 渐进式：先用开发平台，后续再加评测和生态
  • 计费：每个子产品独立计费
  • 适合：初创团队、单一需求场景、预算有限

部署架构

SaaS 模式与子产品模式的部署架构差异。SaaS 共享基础设施，子产品可独立部署。

SaaS 部署 多租户共享

  • Namespace 隔离：每个租户独立 K8S Namespace
  • 共享 LLM Gateway：按租户限流和计量
  • 多区域部署：就近接入，低延迟
  • 弹性伸缩：HPA + VPA 自动扩缩

子产品部署 独立交付

  • Helm Chart：一条命令安装完整子产品
  • Air-Gap：支持离线环境部署
  • 本地模型：可对接 vLLM/Ollama，数据不出域
  • 白标：可替换 Logo、域名、配色
